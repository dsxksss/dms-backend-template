# ADR-0001: 采用 ADR 记录架构决策

- 状态: 已接受
- 日期: 2026-06-11

## 背景

这是一个长期演进、会被多个项目复用的企业级模板，强调**可追溯**。架构上的重要
取舍若只存在于聊天记录或个人记忆中，半年后无人说得清"当初为什么这么选"，
迭代时容易推翻又踩回老坑。

## 决策

所有有长期影响的架构决策都以 ADR（Architecture Decision Record）形式写入
`docs/adr/`，编号递增、只追加不删除；被取代的决策标记为"被取代"并链接到新 ADR。
格式见 [template.md](template.md)。

## 影响

- 正面：决策可追溯，新成员可快速理解"为什么是现在这样"；与 CHANGELOG、
  只追加的 SQL 迁移共同构成全栈可追溯链。
- 成本：每个重要决策需额外几分钟书面记录。
